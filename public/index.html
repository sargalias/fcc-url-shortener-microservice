URL Shortener Microservice

User stories

  1. I can pass a URL as a parameter and I will receive a shortened URL in the JSON response.
  2. If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the JSON response will contain an error instead.
  3. When I visit that shortened URL, it will redirect me to my original link.

Notes

Of course, the URL isn't actually shortened as it is prepended with a very long domain name.

But service works, it creates a short code which it uses to redirect to the original URL.

Instructions

Visit the domain (https://sargalias-url-shortener-ms.herokuapp.com/api/new/YOUR_URL_HERE) replacing YOUR_URL_HERE with a valid URL, including the protocol (such as http://).

You'll be shown a shortenedURL. Visiting it will redirect to your original link.

Example input

https://sargalias-url-shortener-ms.herokuapp.com/api/new/https://google.com

Example output

{
 "shortenedUrl": "https://sargalias-url-shortener-ms.herokuapp.com/api/Hk69kC8oM",
 "originalUrl": "https://google.com"
}
